一、前置环境准备（必装依赖 📦）

Hexo 运行依赖 Git 和 Node.js 环境，需先完成以下两项安装，否则后续命令无法执行：

1. 下载并安装 Git

✅ 作用：用于版本控制和后续博客部署，Hexo 部分功能依赖 Git 环境。

📥 下载地址：Git 官方网站（根据系统选择 Windows/macOS 版本，安装时默认选项即可，无需额外配置）。

2. 下载并安装 Node.js

✅ 作用：提供 npm 包管理器，用于安装和管理 Hexo 及相关依赖。

📥 下载地址：Node.js 官方网站（推荐下载 LTS 长期支持版本，兼容性更好，安装时勾选“Add to PATH”选项）。

✅ 验证安装：安装完成后，打开命令提示符（Windows 按 Win+R 输入 cmd，macOS 打开终端），执行以下命令检查版本：

若终端显示类似 v20.10.0 的版本号，说明 Node.js 安装成功；若提示“不是内部或外部命令”，需重新安装并确保勾选“Add to PATH”。

二、快速搭建 Hexo 博客（核心步骤 🏗️）

通过以下 5 步即可完成本地 Hexo 博客搭建，全程无需复杂配置，新手轻松上手：

1. 全局安装 Hexo 命令行工具

✅ 说明：借助 Node.js 的包管理器（npm）全局安装 Hexo 工具， -g 标志确保在电脑任意目录都能使用 hexo 命令，是后续操作的基础。

2. 初始化博客项目

✅ 说明：创建名为 blog 的博客文件夹（可自定义名称，如 my-hexo-blog），并自动下载 Hexo 基础框架文件，一步完成项目搭建。

3. 进入博客目录

✅ 说明：切换到刚创建的博客文件夹，后续所有 Hexo 命令操作均需在此目录下执行，否则会提示命令无效。

4. 安装依赖包

✅ 说明：安装 Hexo 运行必需的依赖组件，这些组件是博客功能（如渲染、服务器）正常启用的关键，不可跳过。

5. 启动本地服务器

✅ 说明：启动本地预览服务器，浏览器访问 http://localhost:4000 即可查看博客效果，修改内容后页面会实时刷新，方便即时调整。

二、解决 npm 下载慢的实用技巧（提速必备 🚀）

国内网络环境下，npm 默认源（国外服务器）下载速度较慢，推荐以下两种提速方案（二选一即可，效果显著）：

方案一：使用 cnpm 替代 npm

cnpm 是淘宝镜像定制版 npm，专为国内用户优化，下载速度大幅提升：

  1. 安装 cnpm（仅需执行一次）

     `npm install -g cnpm --registry=https://registry.npmmirror.com`

  2. 后续安装操作替换为 cnpm，例如：

     `cnpm install hexo-cli -g  # 替代 npm install hexo-cli -g

    cnpm install # 替代 npm install`

四、博客配置与常用指令说明（高效操作 💡）

1. 文章头部配置含义（Front-matter）

每篇文章顶部的配置项（以 — 包裹）用于定义文章属性，是 Hexo 识别文章的关键，核心参数说明如下：

配置项 含义 示例/说明
title 文章标题 title: Hexo 博客搭建指南
tags 文章标签（支持多个，用逗号分隔） tags: Hexo、教程、博客
categories 文章分类（建议单一分类，逻辑更清晰） categories: 技术教程
date 文章发布时间 自动生成，可手动修改格式 YYYY-MM-DD HH:mm:ss
description 文章简介（用于博客列表页展示） description: 新手入门 Hexo 教程
mathjax 是否启用公式渲染（需数学公式时开启） true 启用 / false 禁用（默认）

2. 高频使用的 Hexo 指令

掌握以下基础指令，即可满足日常博客管理需求，建议收藏备用：

指令 简写 功能说明
hexo clean hexo cl 清除本地生成的静态文件（解决修改不生效、缓存问题）
hexo generate hexo g 生成静态博客文件（部署到网络前必须执行）
hexo server hexo s 启动本地预览服务器（日常编辑必用）
hexo deploy hexo d 部署博客到远程平台（如 GitHub Pages、Coding）
hexo new post 文章名 - 创建新文章（如 hexo new 我的第一篇Hexo博客）

3. 常用指令组合（提高效率）

通过 && 连接指令，实现一步执行多个操作：

  • 本地预览（修改后刷新）：hexo cl && hexo g && hexo s📝 逻辑：先清除缓存 → 生成新文件 → 启动服务器，确保预览内容为最新版本

  • 部署到远程平台：hexo cl && hexo g && hexo d📝 逻辑：先清除缓存 → 生成新文件 → 部署到网络，避免旧内容残留

  • 文章路径：blog/source/_posts/（新建的文章会以 .md 格式保存在此目录，文件名与文章名一致）

  • 编辑工具：可使用任意 Markdown 编辑器（如 Typora、VS Code、Obsidian 等），支持富文本格式编辑

  • 编辑规范：文章内容支持 Markdown 语法，可添加图片、代码块、表格等格式，修改后保存，刷新本地预览页面即可看到效果。
